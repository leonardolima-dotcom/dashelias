04 — Especificações técnicas
Os três estados · dados · lógica
Estado 1 · ListaManter + pequenas melhorias

Lista de Turmas — melhorias na visualização

Melhorias na turma row

  • 3ª linha: "Começa em X dias" calculado dinamicamenteDate.now() → turma.inicio · cor: verde >30d · âmbar 7–30d · vermelho <7d
  • Hover: borda âmbar + fundo leve (já indicado no mockup)
  • Cursor pointer + animação de foco para indicar clicabilidade
  • Badge pulsante quando vagas <5: "Quase lotada!"

Botão Nova Turma

  • Card com borda pontilhada verde no final da listaLinha dashed · fundo rgba(52,211,153,0.04)
  • Texto: "+ Nova Turma · Definir produto, datas, horários e vagas"
  • Ao clicar: transição para Estado 3 (formulário de criação)

Estado (useState)

  • selTurma: null (lista) | turma.id (detalhe) | 'nova' (criação)
  • Ao clicar turma: selTurma = turma.id → renderiza Estado 2
  • Ao clicar "Nova Turma": selTurma = 'nova' → renderiza Estado 3
  • Botão "← Turmas": selTurma = null → volta para lista
Estado 2 · DetalheP1

Detalhe da Turma — Calendário + Alunos + Financeiro

Calendário mensal

  • SVG ou grid CSS de 7 colunas (DOM→SÁB)
  • Células coloridas: turma.inicio → turma.fim em verdeCélula de início: borda esquerda arredondada + dot verde · Célula de fim: borda direita + dot âmbar
  • Navegação ←→ por mês (useState: mesAtual)
  • Hoje: borda âmbar suave
  • Dias passados: opacidade 35%
  • Ao navegar para mês seguinte/anterior: calendário atualiza inline

Seção de horários

  • Lista de dias com horário início → fim + duração calculada
  • Carga horária total calculada automaticamente: soma das durações
  • Tipo por dia: Teórico · Prático · Avaliação (campo de texto livre)
  • Dados de horário em TURMAS.horarios[] (novo campo) ou TURMAS.dias[]{ dia: 'seg', inicio: '08:00', fim: '12:00', tipo: 'Teórico' }

Painel lateral — Alunos

  • Lista de alunos matriculados nessa turma (ALUNOS filtrado por turma)
  • Mini busca inline no painel
  • Badge status pagamento por aluno (paga/parcial/atrasada)
  • Clicar no aluno: navega para tela Base de Alunos com esse aluno selecionado
  • Resumo financeiro da turma: receita esperada · já recebido · em atraso · inadimplência%
  • Barra de progresso de ocupação (confimados ÷ total)
Estado 3 · Nova TurmaP1

Formulário de Criação + Lista de Espera

Formulário (esquerda)

  • Produto: select com produtos cadastrados (PRODUTOS[])Ao mudar produto → lista de espera direita atualiza filtrando por produto
  • Nome da turma: auto-preenchido "Turma N+1" (último id + 1)
  • Local, data início, data fim
  • Total de vagas + vagas bloqueadas
  • Seletor de dias: 7 botões DOM–SÁB, toggle on/off com highlight âmbar
  • Horários por dia: aparece uma linha por dia selecionado com campos início/fim/tipo
  • Carga horária calculada em tempo real
  • Calendário de preview: atualiza em tempo real conforme usuário preenche datas

Lista de espera (direita)

  • Filtra LISTA_ESPERA[] pelo produto selecionado no form
  • Ordenada por: data de entrada na fila (FIFO)
  • Cada item: posição na fila · avatar iniciais · nome · canal de origem · data de entrada
  • Checkbox por aluno: selecionado = será notificado ao criar turma
  • Botão "Selecionar todos" / "Limpar seleção"
  • Resumo no rodapé: "X alunos serão notificados · Agente SDR em 2min"

Ao criar a turma

  • Salvar nova entrada em TURMAS com todos os campos
  • Para cada aluno selecionado: Agente SDR dispara mensagem de convite via WhatsAppTemplate: "Sua vaga na Turma [X] está disponível! Datas: [Y]. Confirme sua inscrição."
  • Disparo automático em até 2 minutos via n8n webhook
  • Remover aluno da LISTA_ESPERA após confirmação de inscrição
  • Atualizar KPIs da tela (turmas abertas, vagas livres)
  • Redirecionar para Estado 1 com nova turma visível no topo

05 — Ordem de implementação
01

Estado 2: Detalhe da Turma — calendário + lista de alunos

Adicionar useState selTurma. Ao clicar numa turma, trocar o render para o layout de detalhe. O calendário usa a data turma.inicio e turma.fim para colorir células. Lista de alunos: filtrar ALUNOS por campo turma (já existe). Financeiro: calcular de ALUNOS.faturamento × ALUNOS.parcelas. Não requer nenhum dataset novo — só filtragem e um novo componente de calendário.

P1Sem novo datasetuseState selTurma · componente CalendarioMes() · filtrar ALUNOS por turma
02

Expandir TURMAS.horarios[] + Seção de horários no detalhe

Adicionar array horarios[] ao dataset TURMAS com os horários de cada dia. Popular os dados das 4 turmas existentes (47–50). Renderizar na tela de detalhe. Carga horária calculada automaticamente. Este passo completa o Estado 2 com a grade de horários — a informação mais útil após as datas.

P1Expandir TURMASAdicionar horarios[] ao TURMAS · calcular cargaHorariaTotal
03

Estado 3: Formulário de criação de nova turma + preview calendário

Formulário com produto, datas, dias da semana e horários. Calendário de preview reativo às datas inseridas. Salvar nova turma em TURMAS. Criar dataset PRODUTOS com os produtos disponíveis para alimentar o select. Sem lista de espera ainda — só criar a turma e redirecionar para lista.

P1Novo dataset PRODUTOSFormulário controlado · calendário preview reativo · salvar em TURMAS
04

Lista de espera + notificação automática via Agente SDR

Criar dataset LISTA_ESPERA. Popular com alunos que entraram em contato mas não havia vaga. Exibir no Estado 3 filtrada por produto. Checkboxes de seleção. Ao criar turma: n8n webhook dispara o Agente SDR para notificar os alunos selecionados com o template de convite. Atualizar status na LISTA_ESPERA após resposta.

P1Novo LISTA_ESPERAn8n webhook → Agente SDR → WhatsApp · atualizar status após confirmação
