04 — Especificações técnicas
Estrutura das 4 abas · dados necessários · lógica
Aba 1 · ResumoP1

Visão Rápida — "O que preciso saber em 5 segundos"

Primeira aba ao selecionar o aluno. Concentra os indicadores mais críticos: saúde financeira, engajamento e próxima ação recomendada. Projetada para o ritual matinal do Seu Elias.

Seções

  • Score de saúde geral (0–100) com anel coloridoVerde ≥70 · Âmbar 40–69 · Vermelho <40
  • 3 indicadores inline: engajamento · pagamento · risco churn
  • Próxima ação recomendada: "Ligar · Enviar lembrete · Reagendar mentoria"Gerada pelo Agente Pós-venda com base nos dados
  • Últimos 3 eventos da timeline (preview)
  • Botão "Ver jornada completa" → muda para aba Jornada

Score de saúde — cálculo

  • Engajamento (40%): horas de tela + módulos concluídos + dias desde último loginPenalizar: sem login >7d = −15pts · sem login >14d = −25pts
  • Financeiro (40%): % de parcelas pagas + dias de atrasoAtraso >15d = −20pts · 2+ parcelas atrasadas = −35pts
  • Engajamento com conteúdo (20%): perguntas · feedback · mentoria agendada
  • Score calculado em tempo real — atualiza ao abrir o painel

Próxima ação recomendada

  • Lógica baseada no score e nos eventos recentes
  • Score <40 + sem login >7d = "Ligar pessoalmente — risco de desistência"Com botão de ligação direta pelo celular
  • Parcela em atraso + sem resposta = "Escalar cobrança para humano"
  • Módulo em progresso há >5d = "Enviar dica motivacional" → template pré-pronto
  • Mentoria disponível e não agendada = "Sugerir agendamento"
Aba 2 · JornadaP1

Engajamento + Módulos + Timeline Cronológica

O coração do painel. Tudo que o aluno fez, em ordem cronológica, com filtro por tipo de evento.

KPIs de engajamento

  • Horas de tela total acumulada na área do alunoSoma de todas as sessões · Fonte: LMS / área do aluno
  • Módulos concluídos / total do produtoEx: 2/12 · com barra de progresso
  • Dias desde último login (com alerta visual se >7d)
  • Streak máximo atingido (dias consecutivos de acesso)
  • Média de minutos por sessão
  • Total de mentorias realizadas

Módulos — detalhe

  • Lista todos os módulos do produto comprado
  • Status: ✓ Concluído · ▶ Em progresso · 🔒 Bloqueado · ○ Disponível
  • Concluído: data de conclusão + tempo total consumido
  • Em progresso: aula atual (X/Y) + último acesso + % completo
  • Click no módulo: expande lista de aulas com status individual
  • Aula travada (parou no meio): badge "Parado aqui" em âmbar

Timeline — tipos de evento

  • 🎓 Matrícula — produto, canal, closer, ciclo de venda, forma de pagamento
  • 💰 Pagamento — parcela número, valor, status, canal (Pix/cartão)
  • ⚠ Atraso — dias em atraso, tentativas de cobrança, resposta do aluno
  • ▶ Acesso — data, módulo acessado, duração da sessão
  • ✓ Módulo concluído — módulo, tempo total, data
  • 🎯 Mentoria — data, duração, tema, feedback (★), link gravação
  • 🤖 Ação do agente — qual agente, canal, mensagem enviada, resposta
  • 📝 Nota manual — adicionada pelo Elias manualmente
  • 🏆 Certificado emitido
Aba 3 · FinanceiroP1

Inadimplência · Parcelas · Score de Recuperação

Expansão do que já existe (grade de parcelas) com score de saúde financeira, histórico de cobranças por parcela e projeção de recuperação.

Dados financeiros

  • Score de saúde financeira (0–100) com anelVerde = paga em dia · Âmbar = parcial/atraso leve · Vermelho = inadimplente
  • Resumo: total pago · em atraso · a vencer · % do total quitado
  • Grade visual de parcelas expandida (mantida do atual)+ tooltip por parcela: data vencimento, data pagamento, canal
  • Próxima parcela: data + valor + dias para vencer (ou dias em atraso)

Histórico de cobranças

  • Por parcela em atraso: tabela de todas as tentativas
  • Colunas: data · mensagem (preview) · canal · status resposta
  • Status de resposta: Pago ✓ · Respondeu · Visto · Não visto · Sem resposta
  • Qual agente enviou vs humanoBadge "🤖 IA" ou "Humano" em cada tentativa
  • Botão "Iniciar negociação" → abre WhatsApp com template

Score de recuperação

  • % de probabilidade de recuperar a dívida atualBaseado em: dias em atraso · histórico de pagamentos · engajamento no curso · perfil
  • Verde >70%: alta chance, manter cobrança automática
  • Âmbar 40–70%: atenção, sugerir negociação direta
  • Vermelho <40%: baixa chance, escalar para Seu Elias / considerar suspensão
  • Receita esperada = % × valor em aberto

Log completo de todas as interações com o aluno: mensagens dos agentes de IA, contatos manuais, respostas recebidas e notas internas adicionadas pelo Seu Elias.

Log de mensagens

  • Todas as mensagens enviadas ao aluno, em ordem cronológica
  • Filtro: Todos · Agente SDR · Agente Cobrança · Agente Pós-venda · Humano · Notas

  • Categorias: Financeiro · Engajamento · Mentoria · Outros

05 — Ordem de implementação
01

Score + 3ª linha na lista + Aba Financeiro expandida

Começa pelos dados que já existem no ALUNOS (parcelas, status_pag, faturamento). Calcular scoreEngajamento e scorePagamento. Adicionar 3ª linha na lista com info contextual. Expandir o painel de Financeiro com grade de parcelas melhorada, score de saúde e histórico de cobranças. Não requer integração com LMS ainda.

P1Dados já existemALUNOS + lógica de score · sem nova integração externa
02

Integração LMS → ALUNO_ENGAJAMENTO + ALUNO_MODULOS

Conectar a plataforma de cursos (Hotmart / Kiwify / própria) ao Supabase via webhook ou polling diário no n8n. Popular ALUNO_ENGAJAMENTO e ALUNO_MODULOS. Só após isso a Aba Jornada fica funcional com dados reais de horas de tela e módulos. Esta é a integração mais crítica — sem ela, a timeline tem apenas dados financeiros.

P1Requer integração LMSn8n workflow → LMS API → Supabase ALUNO_ENGAJAMENTO + ALUNO_MODULOS
03

Aba Jornada: KPIs de engajamento + lista de módulos + timeline

Com os dados do LMS disponíveis, construir a Aba Jornada completa: KPIs de engajamento, barra de progresso por curso, lista de módulos com status e detalhe de aulas, e a timeline cronológica unificando todos os eventos. ALUNO_TIMELINE centraliza tudo — é o aggregator de eventos de todas as fontes.

P1Depende do passo 2 · ALUNO_TIMELINE unifica eventos de todas as fontes
04

Aba Resumo: score geral + próxima ação recomendada + 4 filtros na lista

Score geral ponderando engajamento e financeiro. Lógica de próxima ação recomendada. Filtros na lista (por status, engajamento, produto, ordenação). Ícones de alerta na lista (⚠ 🤖 🎯 ✓). Esta aba depende de todos os dados anteriores estarem disponíveis.

P1Depende dos passos 1 e 2 · lógica de score + UX de filtros
05

Aba Histórico: log GHL + notas manuais + campo de nota inline

Integração com GHL API para buscar histórico de WhatsApp por telefone. Campo para o Elias adicionar notas manuais com categoria. Log de execuções dos agentes filtrado por aluno. Notas injetadas no contexto dos agentes ao interagir com esse aluno.
